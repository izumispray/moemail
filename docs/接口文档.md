## 认证方式

### 方式一：Session 认证

通过 Web UI 登录后自动携带 Session Cookie，适用于浏览器访问。

### 方式二：API Key 认证

在请求头中携带 API Key，适用于程序化调用：

```
X-API-Key: mk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

API Key 可在管理面板中创建，有效期为 1 年。

## 邮箱管理

### 创建邮箱

创建一个新的临时邮箱地址。

```
POST /api/emails/generate
```

**请求体：**

```json
{
  "name": "myemail",        // 可选，邮箱名称（不传则随机生成）
  "domain": "example.com",  // 必填，使用的域名
  "expiryTime": 3600000     // 必填，过期时间（毫秒），0 = 永不过期
}
```

**过期时间选项：**

| 值 | 含义 |
|---|------|
| `600000` | 10 分钟 |
| `3600000` | 1 小时 |
| `86400000` | 24 小时 |
| `604800000` | 7 天 |
| `2592000000` | 30 天 |
| `0` | 永不过期 |

**响应：**

```json
{
  "id": "uuid",
  "email": "myemail@example.com"
}
```

**错误响应：**

| 状态码 | 说明 |
|--------|------|
| `403` | 已达到最大邮箱数量限制 |
| `400` | 无效的过期时间或域名 |
| `409` | 邮箱地址已被使用 |

---

### 获取邮箱列表

获取当前用户的所有活跃邮箱（支持游标分页）。

```
GET /api/emails?cursor={cursor}
```

**查询参数：**

| 参数 | 说明 |
|------|------|
| `cursor` | 可选，分页游标（从上一次响应的 `nextCursor` 获取） |

**响应：**

```json
{
  "emails": [
    {
      "id": "uuid",
      "address": "user@example.com",
      "createdAt": "2024-01-01T00:00:00.000Z",
      "expiresAt": "2024-01-02T00:00:00.000Z",
      "userId": "user-id"
    }
  ],
  "nextCursor": "base64-encoded-cursor",
  "total": 42
}
```

---

### 删除邮箱

```
DELETE /api/emails/{id}
```

**响应：**

```json
{ "success": true }
```

## 子域名邮箱（自动化）

> ⚠️ 仅 **皇帝** 角色可使用。

自动化创建子域名 + DNS 记录 + Email Routing + 邮箱，一步到位。

### 创建子域名邮箱

```
POST /api/emails/subdomain
```

**请求体：**

```json
{
  "domain": "moyii.de",     // 必填，基础域名（需在 KV EMAIL_DOMAINS 中）
  "prefix": "newsletter",   // 可选，子域名前缀（不传则随机生成 6 位）
  "name": "hello",          // 可选，邮箱名称（不传则随机生成 8 位）
  "expiryTime": 86400000    // 可选，邮箱过期时间（毫秒），0 或不传 = 永不过期
}
```

**expiryTime 选项：**

| 值 | 含义 |
|---|------|
| `600000` | 10 分钟 |
| `3600000` | 1 小时 |
| `86400000` | 24 小时 |
| `604800000` | 7 天 |
| `2592000000` | 30 天 |
| `0` 或不传 | 永不过期 |

> 注意：`expiryTime` 仅影响随子域名一并创建的邮箱。子域名本身不会过期，需通过 DELETE 接口手动删除。

**处理流程：**

```
1. 生成子域名前缀 → newsletter.moyii.de
2. 调用 DNS Worker 创建 MX + SPF 记录
3. DNS Worker 启用 Email Routing
4. DNS Worker 设置 catch-all 规则 → Email Worker
5. 子域名追加到 KV EMAIL_DOMAINS（供 generate API 使用）
6. D1 保存 domain 记录
7. D1 创建 email 记录
```

**响应（201）：**

```json
{
  "email": "hello@newsletter.moyii.de",
  "emailId": "uuid",
  "subdomain": "newsletter.moyii.de",
  "domainId": "uuid"
}
```

**错误响应：**

| 状态码 | 说明 |
|--------|------|
| `400` | 缺少 domain 参数 / 域名不在允许列表 / 未配置 Zone ID / 格式不正确 |
| `401` | 未登录或 API Key 无效 |
| `403` | 非皇帝角色 |
| `409` | 子域名已存在 / 邮箱地址已被使用 |
| `500` | DNS Worker 未配置 |
| `502` | DNS 记录创建失败 |

### 删除子域名

删除整个子域名及其所有邮箱，并清理 DNS 记录。

```
DELETE /api/emails/subdomain
```

**请求体：**

```json
{
  "domainId": "uuid"
}
```

**响应：**

```json
{
  "success": true,
  "message": "子域名 newsletter.moyii.de 及其 3 个邮箱已删除",
  "deletedEmails": 3
}
```

## 消息查询

### 获取邮箱消息

获取指定邮箱的收件或发件消息列表（游标分页）。

```
GET /api/emails/{id}?type={type}&cursor={cursor}
```

**查询参数：**

| 参数 | 说明 |
|------|------|
| `type` | 可选，`sent` = 已发送消息，不传 = 已接收消息 |
| `cursor` | 可选，分页游标 |

**响应：**

```json
{
  "messages": [
    {
      "id": "uuid",
      "from_address": "sender@gmail.com",
      "to_address": null,
      "subject": "邮件主题",
      "content": "纯文本内容",
      "html": "<p>HTML 内容</p>",
      "sent_at": null,
      "received_at": 1704067200000
    }
  ],
  "nextCursor": "cursor-string",
  "total": 15
}
```

## 站点配置

> ⚠️ 需要 `MANAGE_CONFIG` 权限。

### 获取配置

```
GET /api/config
```

**响应：**

```json
{
  "defaultRole": "civilian",
  "emailDomains": "example.com,mail.example.com",
  "adminContact": "admin@example.com",
  "maxEmails": "5",
  "domainZones": {
    "example.com": "zone-id-1",
    "mail.example.com": "zone-id-2"
  },
  "turnstile": {
    "enabled": false,
    "siteKey": "",
    "secretKey": ""
  }
}
```

> `turnstile` 字段仅有 `MANAGE_CONFIG` 权限时返回。

---

### 更新配置

```
POST /api/config
```

**请求体：**

```json
{
  "defaultRole": "civilian",
  "emailDomains": "example.com,mail.example.com",
  "adminContact": "admin@example.com",
  "maxEmails": "10",
  "domainZones": {
    "example.com": "zone-id-1"
  },
  "turnstile": {
    "enabled": true,
    "siteKey": "0x...",
    "secretKey": "0x..."
  }
}
```

**配置说明：**

| 字段 | 说明 |
|------|------|
| `defaultRole` | 新用户默认角色：`duke` / `knight` / `civilian` |
| `emailDomains` | 可用域名列表（逗号分隔） |
| `adminContact` | 管理员联系方式 |
| `maxEmails` | 普通用户最大邮箱数量 |
| `domainZones` | 域名到 Cloudflare Zone ID 的映射 |
| `turnstile` | Cloudflare Turnstile 人机验证配置 |

## Webhook 配置

### 获取 Webhook 配置

```
GET /api/webhook
```

**响应：**

```json
{
  "enabled": true,
  "url": "https://your-server.com/webhook"
}
```

### 设置 Webhook

```
POST /api/webhook
```

**请求体：**

```json
{
  "url": "https://your-server.com/webhook",
  "enabled": true
}
```

**Webhook 推送格式：**

当新邮件到达时，POST 请求发送到配置的 URL：

```
Headers:
  Content-Type: application/json
  X-Webhook-Event: new_message
```

```json
{
  "emailId": "uuid",
  "messageId": "uuid",
  "fromAddress": "sender@gmail.com",
  "subject": "邮件主题",
  "content": "纯文本内容",
  "html": "<p>HTML 内容</p>",
  "receivedAt": "2024-01-01T00:00:00.000Z",
  "toAddress": "user@example.com"
}
```

> ⚠️ Key 值仅在创建时返回一次，请妥善保存。

## 权限与角色

### 角色体系

| 角色 | 标识 | 权限 |
|------|------|------|
| **皇帝** (Emperor) | `emperor` | 所有权限，包括站点配置、角色管理 |
| **公爵** (Duke) | `duke` | 创建邮箱、发送邮件、API Key 管理 |
| **骑士** (Knight) | `knight` | 创建邮箱、有限发送 |
| **平民** (Civilian) | `civilian` | 仅创建邮箱 |

### 发送权限检查

```
GET /api/emails/send-permission
```

**响应：**

```json
{
  "canSend": true,
  "remaining": 45,
  "limit": 50
}
```

## DNS Worker 内部接口

> ⚠️ 这些接口由 Pages App 内部调用，不对外暴露。需要 `Authorization: Bearer <DNS_WORKER_SECRET>` 认证。

### DNS Worker 基础地址

```
https://<project-name>-dns-worker.<subdomain>.workers.dev
```

---

### 创建子域名 DNS 记录

```
POST /provision
```

**请求体：**

```json
{
  "zoneId": "cloudflare-zone-id",
  "subdomain": "newsletter",
  "rootDomain": "example.com"
}
```

**处理流程：**

1. 创建 3 条 MX 记录（指向 `route1/2/3.mx.cloudflare.net`）
2. 创建 1 条 SPF TXT 记录
3. 调用 `POST /zones/{zoneId}/email/routing/enable` 启用 Email Routing
4. 调用 `PUT /zones/{zoneId}/email/routing/rules/catch_all` 设置 catch-all 规则

**响应：**

```json
{
  "success": true,
  "domain": "newsletter.example.com",
  "mxRecordIds": ["id1", "id2", "id3"],
  "txtRecordId": "id4",
  "emailRoutingEnabled": true,
  "catchAllSet": true
}
```

---

### 删除 DNS 记录

```
POST /deprovision
```

**请求体：**

```json
{
  "zoneId": "cloudflare-zone-id",
  "recordIds": ["id1", "id2", "id3", "id4"]
}
```

**响应：**

```json
{
  "success": true,
  "results": [
    { "id": "id1", "success": true },
    { "id": "id2", "success": true }
  ]
}
```

### 查找 Zone ID

根据域名自动查找对应的 Cloudflare Zone。

```
POST /find-zone
```

**请求体：**

```json
{
  "domain": "sub.example.com"
}
```

**响应：**

```json
{
  "zoneId": "cloudflare-zone-id",
  "zoneName": "example.com"
}
```

## 错误码说明

| 状态码 | 说明 |
|--------|------|
| `200` | 成功 |
| `201` | 创建成功 |
| `400` | 请求参数错误 |
| `401` | 未认证（未登录或 API Key 无效） |
| `403` | 权限不足 |
| `404` | 资源不存在 |
| `409` | 资源冲突（重复创建） |
| `500` | 服务器内部错误 |
| `502` | 上游服务错误（DNS Worker / Cloudflare API 调用失败） |

所有错误响应格式：

```json
{
  "error": "错误描述信息"
}
```
